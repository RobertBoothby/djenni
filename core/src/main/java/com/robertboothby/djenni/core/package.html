The com.robertboothby.djenni.core package provides the low-level abstractions and helper utilities that every other module in Djenni builds upon. It includes:

  • Supplier infrastructure — the {@link com.robertboothby.djenni.core.StreamableSupplier} extension to {@link java.util.function.Supplier} plus adapters such as {@link com.robertboothby.djenni.core.ThreadLocalSupplier}, {@link com.robertboothby.djenni.core.CachingSupplier}, {@link com.robertboothby.djenni.core.RenewableSupplier}, and {@link com.robertboothby.djenni.core.ValueStore}.
  • Bias and concatenation builders — reusable builders like {@link com.robertboothby.djenni.core.ExplicitlyBiassedSupplierBuilder} and {@link com.robertboothby.djenni.core.ConcatenatingStringSupplierBuilder} for composing higher-order suppliers.
  • Utility APIs — {@link com.robertboothby.djenni.core.SupplierHelper} and {@link com.robertboothby.djenni.core.CharacterStrings} expose a catalogue of factory methods, character sets, array helpers, and zoning utilities that support the individual supplier builders.
  • Coordination helpers — {@link com.robertboothby.djenni.core.CachingSupplierRegistry} and {@link com.robertboothby.djenni.core.NullStreamableSupplier} simplify cross-test coordination, deterministic seeding, and defensive defaults.

Together these types offer a consistent DSL for composing complex generation pipelines using immutable suppliers that can be safely shared across tests, threads, and modules.
