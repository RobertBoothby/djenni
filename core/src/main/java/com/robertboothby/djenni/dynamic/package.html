The com.robertboothby.djenni.dynamic package contains the dynamic supplier infrastructure, centred on DynamicSupplierBuilder. The builder inspects constructors and JavaBean setters, captures each argument as a Parameter, and assigns suppliers drawn from DefaultSuppliers so you can generate data objects without writing bespoke builders.

Key features:

  • Automatic constructor selection (or explicit control via useConstructor/useFunction).
  • Getter/setter-based property binding with fluent .property(...) methods.
  • Sentinel UseDefaultValueSupplier that preserves bean defaults for setter-only properties and enables clearProperty/useDefaultValue to revert configuration safely.
  • Extensible default supplier registry (DefaultSuppliers) so teams can define organisation-wide fixtures.

Supporting classes (ParameterContext, IntrospectionParameterContext, RuntimeParameterContext) capture parameter metadata during introspection and replay it with the correct suppliers at runtime.

Built StreamableSupplier instances are immutable snapshots of the builder configuration. You can keep using an existing supplier while reconfiguring the same DynamicSupplierBuilder to build another supplier without having to re-run introspection.

See README.md in this package for a full usage guide and extended examples.
